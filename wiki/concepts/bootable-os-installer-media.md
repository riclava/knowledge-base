Bootable OS installer media 是把完整安装器写入外部介质，使系统能够在现有操作系统之外执行安装、重装、降级或恢复的操作模式；它强调“拿到安装器”和“真正能从外部介质启动安装环境”是两个不同阶段。

## Definition

在当前知识库里，这个概念首先来自 macOS 的安装 U 盘制作流程，但它抽象出的并不是某一个命令，而是一条更通用的安装介质准备链路：

- 先确认目标版本和目标机器的兼容关系
- 再准备外部介质并进行格式化
- 获取完整安装器，而不是只拿到一个更新入口
- 使用平台提供的工具把安装器写成可启动介质
- 通过启动选择器进入安装环境
- 如果设备对外部启动有策略限制，再进入恢复/安全控制面解除阻塞

## Generic Workflow

| 阶段 | 目标 | 典型动作 |
|---|---|---|
| 兼容性确认 | 确保目标版本可装 | 查询支持机型或版本边界 |
| 介质准备 | 提供可写入的外部存储 | 选择 U 盘 / 外部盘并按要求格式化 |
| 安装器获取 | 拿到完整安装资源 | 下载完整安装器应用或镜像 |
| 介质制作 | 生成可启动结构 | 使用官方或平台提供的制作工具 |
| 启动选择 | 进入安装环境 | 通过启动选择器选择外部安装盘 |
| 安装执行 | 执行重装/升级/降级 | 进入安装程序并选择目标磁盘 |
| 结果验证 | 确认变更真的生效 | 检查系统版本、启动盘和基础功能 |

## Operational Surfaces

| 操作面 | 要回答的问题 |
|---|---|
| 版本兼容性 | 目标系统版本是否真的支持目标硬件 |
| 介质格式 | 外部存储的文件系统、分区方案和卷名是否满足制作工具要求 |
| 安装器形态 | 拿到的是完整安装器，还是只是一个更新入口或下载器 |
| 制作工具 | 应该用哪个官方命令或工具写入安装介质 |
| 启动入口 | 设备应通过什么按键、菜单或固件入口选择外部介质 |
| 安全策略 | 设备是否允许外部启动，是否需要先进入恢复环境调整策略 |
| 回退手段 | 启动失败后应如何恢复默认启动状态或重置相关参数 |

## Why It Matters

- 它把“重装系统”从一句笼统描述拆成可执行、可验证的多个阶段。
- 它帮助文档作者把兼容性、外部介质准备、启动入口和安全策略分开写清楚。
- 它常用于系统损坏、离线安装、版本降级或跨版本恢复等不适合在当前系统内直接升级的场景。

## Common Misconceptions

| 误解 | 更准确的理解 |
|---|---|
| 下载了安装器就等于安装介质已经准备好 | 还需要把完整安装器写入外部介质并验证可启动性 |
| 把文件复制到 U 盘就能启动安装 | 通常需要专用工具生成可启动结构，而不是普通文件复制 |
| 只要 U 盘做好了，机器就一定能从它启动 | 启动安全策略、兼容性和启动入口都会影响结果 |
| 所有版本的安装器命令路径都相同 | 工具路径和安装器名称往往依赖具体版本 |

## Documentation Implications

- 文档应把“兼容性检查”“介质准备”“安装器获取”“启动进入”“安装后验证”写成独立步骤。
- 应明确区分版本特定参数和通用模式，避免读者照搬错误的安装器路径或卷名。
- 应把外部启动策略、恢复模式和回退方式写入同一条操作链路，而不是留到故障排查时才补充。
- 如果文档面向最终用户，应补充备份、磁盘擦除和恢复后数据迁移的前置说明。
