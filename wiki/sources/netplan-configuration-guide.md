Comprehensive reference for configuring network interfaces on Ubuntu using Netplan, covering YAML syntax, common scenarios, advanced patterns, and troubleshooting.

## Key Points

### What Netplan Is

- Default network configuration tool for Ubuntu 17.10+ and Ubuntu Server 18.04+
- Uses YAML configuration files
- Acts as abstraction layer over two possible backends:
  - `systemd-networkd` (server default)
  - `NetworkManager` (desktop default)

### Configuration File Hierarchy

Priority from lowest to highest:
1. `/lib/netplan/*.yaml` — system defaults
2. `/etc/netplan/*.yaml` — main configuration directory
3. `/run/netplan/*.yaml` — runtime overrides

Common filenames:
- `00-installer-config.yaml` — generated by installer
- `01-netcfg.yaml` — custom configuration

### YAML Syntax Rules

- Indentation-sensitive: must use spaces, not tabs
- 2 spaces per indentation level
- Colon must be followed by a space
- File permissions should be 600 or 644

### Basic Structure

```yaml
network:
  version: 2
  renderer: networkd  # or NetworkManager
  ethernets:
    # Ethernet interfaces
  wifis:
    # WiFi interfaces
  bridges:
    # Bridge interfaces
  bonds:
    # Bond interfaces
  vlans:
    # VLAN interfaces
```

## Configuration Scenarios

### DHCP (Default)

```yaml
network:
  version: 2
  renderer: networkd
  ethernets:
    eth0:
      dhcp4: true
      dhcp6: false
```

### Static IP

```yaml
network:
  version: 2
  renderer: networkd
  ethernets:
    eth0:
      addresses:
        - 192.168.1.100/24
      routes:
        - to: default
          via: 192.168.1.1
      nameservers:
        addresses:
          - 8.8.8.8
          - 114.114.114.114
        search:
          - example.com
```

### Multi-NIC with Separate Routes

```yaml
network:
  version: 2
  ethernets:
    eth0:
      addresses: [192.168.1.100/24]
      routes:
        - to: default
          via: 192.168.1.1
    eth1:
      addresses: [10.0.0.100/24]
      routes:
        - to: 10.0.0.0/8
          via: 10.0.0.1
```

### Static Routes with Metrics

```yaml
routes:
  - to: default
    via: 192.168.1.1
    metric: 100
  - to: 10.0.0.0/8
    via: 192.168.1.254
```

### VLAN

```yaml
network:
  version: 2
  ethernets:
    eth0:
      dhcp4: false
  vlans:
    vlan10:
      id: 10
      link: eth0
      addresses: [192.168.10.100/24]
```

### Bond (Link Aggregation)

```yaml
network:
  version: 2
  ethernets:
    eth0: {dhcp4: false}
    eth1: {dhcp4: false}
  bonds:
    bond0:
      interfaces: [eth0, eth1]
      addresses: [192.168.1.100/24]
      parameters:
        mode: active-backup
        mii-monitor-interval: 100
        primary: eth0
```

Bond modes:
- `balance-rr` (0): Round-robin load balancing
- `active-backup` (1): Failover
- `balance-xor` (2): XOR-based
- `broadcast` (3): Broadcast all
- `802.3ad` (4): LACP (requires switch support)
- `balance-tlb` (5): Transmit load balancing
- `balance-alb` (6): Adaptive load balancing

### Bridge

```yaml
network:
  version: 2
  ethernets:
    eth0: {dhcp4: false}
    eth1: {dhcp4: false}
  bridges:
    br0:
      interfaces: [eth0, eth1]
      addresses: [192.168.1.100/24]
      parameters:
        stp: true
        forward-delay: 4
```

### WiFi

```yaml
network:
  version: 2
  renderer: NetworkManager
  wifis:
    wlan0:
      access-points:
        "SSID名称":
          password: "WiFi密码"
      dhcp4: true
```

### Multiple IP Addresses

```yaml
addresses:
  - 192.168.1.100/24
  - 192.168.1.101/24
  - 192.168.1.102/24
```

### IPv6

```yaml
addresses:
  - 192.168.1.100/24
  - 2001:db8::100/64
routes:
  - to: default
    via: 192.168.1.1
  - to: default
    via: 2001:db8::1
nameservers:
  addresses: [8.8.8.8, 2001:4860:4860::8888]
```

## Advanced Patterns

### Match by Interface Name Pattern

```yaml
ethernets:
  all-ethernet:
    match:
      name: "en*"
    dhcp4: true
```

### Match by MAC Address and Rename

```yaml
ethernets:
  wan:
    match:
      macaddress: "00:11:22:33:44:55"
    set-name: wan0
    addresses: [192.168.1.100/24]
```

### Disable IPv6

```yaml
ethernets:
  eth0:
    dhcp4: true
    dhcp6: false
    ipv6-privacy: false
    link-local: []
```

### Override DHCP Options

```yaml
ethernets:
  eth0:
    dhcp4: true
    dhcp4-overrides:
      use-dns: false
      use-routes: false
    nameservers:
      addresses: [8.8.8.8, 1.1.1.1]
```

### Set MTU

```yaml
ethernets:
  eth0:
    dhcp4: true
    mtu: 1450
```

## Operational Commands

| Command | Purpose |
|---------|---------|
| `sudo netplan try` | Test configuration with 120-second auto-rollback |
| `sudo netplan apply` | Apply configuration |
| `sudo netplan generate` | Generate backend config files without applying |
| `sudo netplan --debug apply` | Apply with debug output |

### Verification Commands

```bash
ip addr show          # View IP addresses
ip link show          # View interface status
ip route show         # View routing table
resolvectl status     # View DNS configuration (or systemd-resolve --status)
```

### Service Management

```bash
# For systemd-networkd backend
sudo systemctl restart systemd-networkd
sudo systemctl status systemd-networkd

# For NetworkManager backend
sudo systemctl restart NetworkManager
```

## Troubleshooting

### Syntax Validation

```bash
sudo netplan --debug generate
sudo journalctl -u systemd-networkd -f
```

### Common Errors

| Error | Cause | Fix |
|-------|-------|-----|
| Invalid YAML | Tabs instead of spaces, missing space after colon, unmatched quotes | Use spaces, check syntax |
| Network unreachable | Wrong gateway, missing route | Verify `ip route show`, check gateway |
| DNS resolution fails | Missing or wrong nameservers | Check `resolvectl status`, verify nameserver addresses |

### Emergency Recovery

If configuration error causes loss of connectivity:

```bash
# Temporary IP assignment (lost on reboot)
sudo ip addr add 192.168.1.100/24 dev eth0
sudo ip link set eth0 up
sudo ip route add default via 192.168.1.1

# Or use dhclient
sudo dhclient eth0
```

### Safe Testing Workflow

1. Edit: `sudo nano /etc/netplan/01-netcfg.yaml`
2. Test: `sudo netplan try` (auto-rollback after 120 seconds if not confirmed)
3. Apply: `sudo netplan apply`

## Caveats and Risks

- **YAML sensitivity**: A single indentation error breaks the entire configuration
- **Backend differences**: Some features only work with specific backends (e.g., WiFi typically requires NetworkManager)
- **No atomic rollback**: Unlike `netplan try`, `netplan apply` has no built-in rollback
- **WiFi passwords in plaintext**: Configuration files may contain sensitive credentials
