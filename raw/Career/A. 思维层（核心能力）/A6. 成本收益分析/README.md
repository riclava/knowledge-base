# A6. 成本收益分析

> 状态：📝 大纲 ｜ 最近更新：2026-06-03

## 是什么 / 解决什么问题

- 成本收益分析（CBA）：系统比较一个决策的总成本与总收益，支撑取舍。
- 要解决的问题：资源有限下"该不该做、先做哪个、做到什么程度"。
- 在技术工作中的位置：技术债偿还、重构投入、自研 vs 采购、容量投入。

## 核心原理

- 显性成本 vs 隐性成本（机会成本、维护成本、认知负担）。
- 沉没成本：已花费且不可回收，决策时应忽略。
- 边际分析：边际成本 vs 边际收益，做到二者相等为最优。
- 时间价值：折现、TCO（总拥有成本）、ROI / 回收期。
- 量化与不可量化收益的处理（可靠性、士气、学习曲线）。

## 关键细节 / 易错点

- 沉没成本谬误："都投了这么多了不能停"。
- 只算开发成本，忽略长期运维与维护成本。
- 高估收益、低估成本（规划谬误、乐观偏差）。
- 忽略机会成本：做了 A 就做不了更高价值的 B。
- 把难量化的收益直接当成零。

## 实战与权衡

- 技术债：偿还成本 vs 持续拖累的利息。
- 自研 vs 采购 vs 开源：TCO 与控制力、锁定风险的权衡。
- 性能优化的边际收益递减：何时该停。
- 短期交付 vs 长期可维护性的取舍。
- 与 A7 风险分析结合：期望值 = 收益 × 概率 − 风险敞口。

## 一句话讲清楚

（待补：成本收益分析就是在资源有限时，用"投入产出 + 机会成本 + 忽略沉没成本"判断一件事到底值不值得做。）
