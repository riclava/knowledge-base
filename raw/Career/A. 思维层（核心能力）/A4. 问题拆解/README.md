# A4. 问题拆解

> 状态：📝 大纲 ｜ 最近更新：2026-06-03

## 是什么 / 解决什么问题

- 问题拆解：把一个大而模糊的问题切成更小、更具体、可独立处理的子问题。
- 要解决的问题：问题太大无从下手、范围不清、责任无法分配、进度不可度量。
- 在技术工作中的位置：需求分析、任务分解（WBS）、故障定位、技术方案设计。

## 核心原理

- MECE 原则：相互独立、完全穷尽。
- 自顶向下分解 vs 自底向上归纳。
- 分治思想：拆 → 解 → 合（与算法中的 Divide & Conquer 同源）。
- 问题树 / 逻辑树 / Issue Tree。
- 抓主要矛盾：二八法则，先解决高价值/高风险子问题。

## 关键细节 / 易错点

- 拆解维度不一致（混用功能维度与流程维度）。
- 子问题相互重叠或有遗漏（违反 MECE）。
- 只拆不合：缺少子结果整合回原问题的步骤。
- 拆解粒度失控：过粗无法执行，过细陷入细节。
- 没有先定义"问题本身"就急着拆（解错了问题）。

## 实战与权衡

- 故障定位：二分法 / 排除法快速缩小范围。
- 任务分解：WBS、用户故事拆分（垂直切片 vs 水平切片）。
- 假设驱动拆解：先提假设，再用拆解验证（咨询常用）。
- 拆解深度的权衡：拆解成本 vs 执行清晰度收益。
- 与 A1 逻辑推理、A2 抽象建模的协同关系。
