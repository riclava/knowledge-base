# A8. 技术决策能力

> 状态：📝 大纲 ｜ 最近更新：2026-06-03

## 是什么 / 解决什么问题

- 技术决策：在多个技术方案间，基于约束与目标做出选择并对结果负责。
- 要解决的问题：信息不全、众说纷纭、影响深远且难以回退时如何选得清楚、可追溯。
- 在技术工作中的位置：架构选型、技术栈选择、Build vs Buy、演进路线。
- 它是前面 A1–A7 思维能力的综合出口。

## 核心原理

- 决策三要素：目标/约束、备选方案、评估标准。
- 决策类型：可逆（单向门）vs 不可逆（双向门），决策速度应不同。
- 权衡矩阵 / 加权打分：多维度对比备选项。
- 满意决策（satisficing）vs 最优决策：信息成本下的现实选择。
- 决策记录：ADR（Architecture Decision Record）固化上下文与理由。

## 关键细节 / 易错点

- 简历驱动开发 / 技术追新，脱离实际场景。
- 锚定单一方案，未认真生成备选项。
- 用"喜好"代替"标准"，评估维度不透明。
- 忽视可逆性：对可回退的决策过度纠结，对不可逆的决策过于草率。
- 决策后不复盘，无法形成反馈与改进。

## 实战与权衡

- 技术选型框架：成熟度、社区、团队熟悉度、运维成本、锁定风险。
- Build vs Buy vs 开源：控制力 vs 成本 vs 速度。
- 一致性 vs 多样性：统一技术栈 vs 因地制宜。
- 短期交付压力 vs 长期架构健康。
- 决策流程：何时独裁、何时共识、何时升级（与 M 技术领导力结合）。

## 一句话讲清楚

（待补：技术决策能力就是在信息不全时，明确目标与约束、列清备选与标准、按可逆性决定快慢，并把理由记录下来供日后检验。）
