# 迭代运转模式

## ⚡️ 小团队极捷协作视图 (1周/迭代)

| 时间 | 周一 | 周二 | 周三 | 周四 | 周五 |
|------|------|------|------|------|------|
| **上午** | ☀️ 站立晨会 (10min) | ☀️ 站立晨会 (10min) | ☀️ 站立晨会 (10min) | ☀️ 站立晨会 (10min) | ☀️ 站立晨会 (10min) |
| | 迭代启动 ⏱15min `PM` | 专心Coding | 需求评审 ⏱60min `PD` | 技术方案讨论 ⏱60min `BE` `FE` | 走查与验收 ⏱60min `PD` |
| **下午** | 项目周报 `PM` | 上线 / 内部灰度发布 `OP` | 专心Coding | 规划下一迭代需求 `PD` | Showcase展示 / 迭代回顾(复盘) ⏱60min `PM` `PD` |

## 规则

- 统一迭代长度：**一周**
- 小团队倡导敏捷，各项活动由 **Owner** 发起并控制时间

## 角色缩写说明 (全栈作战)

| 缩写 | 角色 |
|------|------|
| PM | 项目经理 (把控进度) |
| PD | 产品经理 (需求/验收) |
| FE | 前端开发 |
| BE | 后端开发 |
| OP | 技术运维 (部署/基建) |
