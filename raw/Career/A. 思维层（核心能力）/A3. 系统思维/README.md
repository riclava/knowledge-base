# A3. 系统思维

> 状态：📝 大纲 ｜ 最近更新：2026-06-03

## 是什么 / 解决什么问题

- 系统思维：把对象看作相互关联的整体，而非孤立部件之和。
- 关注要素、连接关系、目标三者，而非只盯单点。
- 要解决的问题：局部最优导致全局变差、"按下葫芦浮起瓢"、副作用与滞后效应。
- 在技术工作中的位置：架构演进、容量规划、组织/系统协同（康威定律）。

## 核心原理

- 反馈回路：增强回路（正反馈）与调节回路（负反馈）。
- 时滞（Delay）：因与果在时间上分离，导致震荡与过度反应。
- 存量与流量（Stock & Flow）：状态积累 vs 速率变化。
- 涌现（Emergence）：整体行为不可由部件简单推出。
- 瓶颈与约束理论（TOC）：系统吞吐由最弱环节决定。

## 关键细节 / 易错点

- 只优化局部、忽略全局（子系统最优 ≠ 系统最优）。
- 忽略反馈时滞，导致过度修正、来回震荡。
- 把症状当根因（治标不治本，问题转移）。
- 线性思维套用到非线性/有反馈的系统上。
- 忽视"政策阻力"：各方对抗使干预失效。

## 实战与权衡

- 容量规划与限流降级：识别瓶颈与反馈回路。
- 故障扩散分析：级联失败、重试风暴、雪崩效应。
- 康威定律：组织结构与系统架构的相互塑造。
- 干预点选择：高杠杆点（规则、目标、范式）vs 低杠杆点（调参数）。
- 短期 vs 长期：局部快速止血 vs 结构性根治的权衡。
