# A7. 风险分析

> 状态：📝 大纲 ｜ 最近更新：2026-06-03

## 是什么 / 解决什么问题

- 风险分析：识别、评估、应对未来可能发生且带来损失的不确定性。
- 要解决的问题：不确定环境下避免被"黑天鹅"击穿，提前留出余量与预案。
- 在技术工作中的位置：架构评审、上线变更、容灾设计、依赖与供应链评估。

## 核心原理

- 风险 = 可能性（概率）× 影响（损失）。
- 风险矩阵：按高/中/低对概率与影响分级排序。
- 四种应对策略：规避、转移、缓解、接受。
- 已知风险 vs 未知风险（已知的未知 / 未知的未知）。
- 尾部风险与黑天鹅：低概率高影响事件的非对称性。

## 关键细节 / 易错点

- 只关注高概率事件，忽略低概率高影响的尾部风险。
- 概率与影响估计的主观偏差（过度自信、近因效应）。
- 风险登记后无人跟踪、无触发条件、无 owner。
- 缓解措施本身引入新风险（如重试导致雪崩）。
- 把"没出过事"当成"没有风险"（幸存者偏差）。

## 实战与权衡

- 变更管理：灰度、回滚预案、爆炸半径控制。
- 架构层面：单点识别、冗余、隔离（舱壁/熔断/限流）。
- FMEA（失效模式与影响分析）、Pre-mortem（事前验尸）。
- 风险成本 vs 防护成本：不是所有风险都值得防（与 A6 结合）。
- 容灾等级（RTO/RPO）与投入的权衡。

（待补：风险分析就是提前问"哪里会出事、概率多大、损失多大、出了怎么办"，把不确定性变成可管理的清单和预案。）
