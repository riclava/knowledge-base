技术过硬只是入场券，专家岗和全职远程更看重"软实力"——你怎么做决策、怎么带人、怎么在没人盯着的情况下高效协作。这一篇按行为面（BQ）高频问题逐一拆解，通用答题框架是 **STAR**（Situation 情境 → Task 任务 → Action 行动 → Result 结果，结果尽量量化）。

### 🌟 讲一个你主导的、最有挑战的核心项目

**关键点**：清晰交代背景和你的角色（主导/核心）、难点在哪、你做了哪些关键决策和权衡、量化结果（QPS 提升 X 倍、P99 从 A 降到 B、成本降 X%）。

🧠 **记忆**：讲项目要突出"**你的**决策和取舍"，而不是流水账；面试官想看的是你的判断力，不是项目本身多牛。

### ⚡ 你做过最复杂的性能优化案例

**关键点**：怎么定位瓶颈（pprof/监控数据说话）、对比了哪几种方案、为什么选这个、优化前后数据对比、有没有副作用/权衡。

🧠 **记忆**：性能优化题的灵魂是"**先测量再优化**"——能说出"我通过 xx 工具发现瓶颈在 yy"比"我觉得这里慢"专业得多。

### 🔥 一次线上重大事故，你如何处理和复盘

**关键点**：冷静先止血、主动承担、系统性改进（不是打补丁）、blameless 复盘、落地了哪些长效改进。

🧠 **记忆**：面试官看的是"你面对压力的处理方式和责任心"，别甩锅、别只讲技术细节忽略协作。（技术细节见 [[线上故障排查与复盘]]。）

### 🤝 技术选型有分歧时如何推进决策

**关键点**：用成本/收益/风险分析、数据和 POC 说话、倾听不同意见、达成共识、把决策和理由记录下来（ADR 架构决策记录）。

🧠 **记忆**：专家不是"我说了算"，而是"用事实和数据让大家心服口服"，并对决策负责、留档可追溯。

### 🧑‍🏫 如何带初级开发做技术传帮带

**关键点**：通过 code review 引导思考（多问"为什么"少直接给答案）、沉淀文档、结对编程、合理拆解任务给成长空间、容忍可控的试错。

🧠 **记忆**：带人不是"帮他把活干了"，而是"让他学会自己干"——授人以渔。

### 🏠 全职远程如何保证自律和高效协作

**关键点**：结果导向（对齐目标和交付）、异步沟通（清晰的文字/文档/记录减少同步依赖）、主动同步进度（不让人追着问）、约定响应时效和重叠时间、可复盘可追溯（重要决策落文字）。

🧠 **记忆**：远程协作的核心是"**主动 + 透明 + 文档化**"——让协作者随时能知道你在做什么、进展如何，不靠"看到你在工位"来建立信任。

### 🔭 如何持续跟进前沿技术并落地业务

**关键点**：稳定的信息源、判断标准（是否解决真实业务痛点，而非追新）、小范围 POC 验证、评估引入成本/风险/团队学习曲线，再决定是否推广。

🧠 **记忆**：专家对新技术是"**冷静评估、小步验证**"而非"看到就上"——技术服务于业务，不是炫技。

### 📌 记忆卡

*   **框架**：STAR + 量化结果 + 突出你的决策和权衡。
*   **项目**：讲你的判断力，不是流水账。
*   **性能优化**：先测量再优化。
*   **事故**：先止血 + 主动担 + 系统性改进 + blameless。
*   **选型分歧**：成本/收益/风险 + 数据 + POC + ADR 留档。
*   **带人**：授人以渔，引导思考。
*   **远程**：主动 + 透明 + 文档化。
*   **前沿技术**：冷静评估、小步验证，服务业务不炫技。
