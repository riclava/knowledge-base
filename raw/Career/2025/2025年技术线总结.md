## AI时代对我们提出的要求

+ 完整交付能力：需要具备完整的软件产品交付流程经验
+ 不断学习：AI时代，需要学习的只会更多，不会更少
+ 更扎实的底座支持：组件化、模块化、BaaS等能力，快速支持AI应用落地
+ 理解AI时代的水和电 = 算力 + 数据 + 基础平台
+ AI 时代：软件行业的「不变」
    + 问题价值不变：还是解决真实业务问题，而不是"炫技术"。
    + 工程本质不变：可靠性、可维护性、可扩展性仍是核心。
    + 成本约束不变：性能、资源、钱，依然是第一性约束。
    + 用户体验不变：好产品仍然以用户价值为中心。
    + 团队能力不变：架构、抽象、协作能力仍然决定上限。
+ AI 时代：软件行业的「变」
    + 软件形态在变：从确定性 → 概率性（传统：规则 + 逻辑，AI时代：模型 + Prompt + 数据）
    + 开发方式在变：从编码为主 → 编码+AI 协同
    + 技术栈在变：新增 AI 基础层，算力、数据、向量库、LLMOps 成为"基础设施"。
    + 产品形态在变：从工具 → 智能体（App = 功能，AI App = 功能 + 智能决策）
    + 组织分工在变：新增 AI 工程角色，AI Engineer / Prompt Engineer / MLOps / 评估工程师
    + 价值来源在变：从效率 → 效率 + 创新，不只是降本增效，还能创造新产品形态。

## 2026工作规划

+ 架构与平台（底座）：可扩展、可复用、可治理
  + 架构收敛，减少烟囱式系统
  + 平台化能力：配置中心、日志监控告警、权限网关
  + AI 基础层：推理服务、向量存储、LLMOps
  + 指标：新系统 70% 复用平台能力，核心服务标准化 ≥ 80%

+ 工程效率（交付）：提效率，缩周期
  + CI/CD 完善：构建 -30%，发布 +50%
  + 自动化测试覆盖率 ≥ 60%
  + AI Coding 助手辅助开发
  + 统一代码风格、接口规范、文档标准
  + 指标：交付周期 -20~30%，线上缺陷 -30%

+ 稳定性与运维（可靠性）：从"不稳定"到"可预测"
  + 可观测性：日志 + 指标 + Trace
  + 事故管理流程优化：事前、事中、事后
  + 容灾与降级方案建设
  + 指标：核心可用性 ≥ 99.9%，P0/P1 事故 -40%

+ 团队与组织（长期能力）：提能力，去依赖
  + 每季度技术分享
  + 职责清晰化：架构、交付、稳定性分工明确
  + 指标：团队成员具备独立交付项目的能力

+ AI融合：挖潜存量业务潜力
  + 每个项目根据实际情况，融入AI能力
  + 指标：每个项目至少有1-2个功能，完成AI融合且用户活跃度较高
