# A5. 第一性原理

> 状态：📝 大纲 ｜ 最近更新：2026-06-03

## 是什么 / 解决什么问题

- 第一性原理：回到不可再分的基本事实/物理约束出发推理，而非类比已有方案。
- 要解决的问题：路径依赖、"一直都这么做"、被现有方案与行业惯例锁死。
- 在技术工作中的位置：技术选型、性能极限分析、成本结构重构、颠覆式创新。

## 核心原理

- 基本公理/约束：物理定律、数学下界、信息论极限、成本构成。
- 演绎重建：从公理出发重新推导方案，而非归纳/类比。
- 区分"本质约束"与"人为约束"（惯例、历史包袱）。
- 与类比思维的对比：类比快但受限于参照系，第一性慢但能突破。

## 关键细节 / 易错点

- 误把当前实现的限制当成本质约束。
- 滥用：对低价值/已成熟问题做第一性推导，成本极高且无收益。
- 忽略经济与时间约束：理论可行 ≠ 当下值得做。
- 拆到错误的"基本单元"，地基错了推导全错。
- 只破不立：质疑了惯例却给不出可执行的重建方案。

## 实战与权衡

- 性能分析：从硬件极限（带宽、延迟、IOPS）反推理论上界。
- 成本重构：拆解成本到原子项，找可被消除的"非本质"成本。
- 技术选型：剥离营销话术，回到问题本质需求再匹配方案。
- 第一性 vs 类比/最佳实践：高风险高回报的探索 vs 低成本的复用。
- 何时使用：方向性、长期、可能被颠覆的关键决策。
